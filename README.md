# electrolab

Веб‑реестр протоколов испытаний (Flask + SQLite) со сканированием DOCX-файлов из сетевой папки.

## Что делает проект

- Периодически сканирует папку `ROOT_FOLDER` и добавляет новые протоколы в БД.
- Обрабатывает только валидные DOCX-файлы протоколов (игнорирует временные `~$`).
- Извлекает из DOCX структурированные данные:
  - номер протокола;
  - название протокола;
  - объект;
  - дату проведения испытаний;
  - исполнителей.
- Сохраняет путь к файлу, дату изменения, тип испытаний и номер ячейки.
- Показывает записи в веб-интерфейсе с фильтрами и поиском.

## Извлечение данных из DOCX

Парсинг реализован в `scanner.py`:

- `read_docx_text(path)` — читает непустые абзацы из DOCX.
- `parse_protocol_data(text)` — разбирает текст по шаблонам:
  - `протокол №...` → `protocol_number`;
  - следующая строка после строки с «протокол» → `protocol_title`;
  - `объект: ...` / `на объекте ...` → `object_name`;
  - `дата проведения испытаний: ...` → `test_date`;
  - `испытания произвели: ...` → `engineers`.

Чтобы не импортировать один и тот же файл повторно, используется проверка `protocol_exists(path)` по `file_path`.

## Структура БД

Таблица `protocols` включает ключевые поля:

- `protocol_number`
- `protocol_name`
- `object_code`
- `object_name`
- `protocol_title`
- `test_date`
- `engineers`
- `test_type`
- `cell_number`
- `content_text`
- `file_path` (уникальное)
- `modified_date`

`init_db()` автоматически добавляет недостающие колонки через `ALTER TABLE` для уже существующих БД.

## Фильтры в интерфейсе

На главной странице доступны:

- полнотекстовый поиск (`search`);
- фильтр по объекту (`object_filter`);
- фильтр по типу испытаний (`type_filter`);
- фильтр по году (`year`) по префиксу номера протокола:
  - пример: `26-С3-15` → год `26` (2026),
  - SQL-условие: `protocol_number LIKE '26-%'`.

## Запуск

1. Установить зависимости:

   ```bash
   pip install flask apscheduler python-docx
   ```

2. Проверить путь сканирования в `scanner.py`:
   - `ROOT_FOLDER = r'\\Admin\рабочая'`

3. Запустить приложение:

   ```bash
   python app.py
   ```

После запуска:

- создаётся/обновляется `protocols.db`;
- стартует фоновый планировщик (`scan_folders` каждые 5 минут);
- веб-интерфейс доступен на локальном Flask-адресе.
